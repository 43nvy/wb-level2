Что выведет программа? Объяснить вывод программы. Объяснить как работают defer’ы и их порядок вызовов.

```go
package main

import (
	"fmt"
)


func test() (x int) {
	defer func() {
		x++
	}()
	x = 1
	return
}


func anotherTest() int {
	var x int
	defer func() {
		x++
	}()
	x = 1
	return x
}


func main() {
	fmt.Println(test())
	fmt.Println(anotherTest())
}
```

Ответ:
```
Вывод: 2\n1\n
Так происходит из за особенностей defer:
defer - откладываем выполнение функции, пока окружающая функция не завершится.
Отложенные функции находятся в той же зоне видимости что и окружающая функция, поэтому видят и ее изменения.
Работает по принципо LIFO (last-in-first-out) - последним пришел - первым ушел.
Теперь, мы можем обьяснить почему сначала вывелось 2, а потом 1:
В функции теле функции test мы устанавливаем значение "х" = 1, а затем, срабатывает отложенная функция, которая добавляет еще 1 и возвращает результат в функцию test,
а test, в свою очередь, возвращает значение в main, где оно и выводиться.
Затем вызывается функция anotherTest, мы обьявляем переменную "x", затем присваиваем значение = 1, и возвращаем его в main, а потом уже срабатывает анонимная функция в теле anotherTest, которая добавляет 1 к х. Тоесть main функция уже получила 1 и вывела его.

```

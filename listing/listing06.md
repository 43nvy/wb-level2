Что выведет программа? Объяснить вывод программы. Рассказать про внутреннее устройство слайсов и что происходит при передачи их в качестве аргументов функции.

```go
package main

import (
	"fmt"
)

func main() {
	var s = []string{"1", "2", "3"}
	modifySlice(s)
	fmt.Println(s)
}

func modifySlice(i []string) {
	i[0] = "3"
	i = append(i, "4")
	i[1] = "5"
	i = append(i, "6")
}
```

Ответ:
```
Вывод: [3 2 3]
Тут дело в подкапотке слайсов. Когда мы инициализируем слайс - под капотом создается массив. Слайс получает указатель на первый элемент, использующийся в слайсе,
этого массива,а последний элмент слайса определяется параметром length.
Также, у слайса есть параметр capacity - он показывает сколько элементов влазит в базовый массив.
В нашем случае, в main функции мы создаем слайс "s", под него создается массив [3]string{"1", "2", "3"}, "s" хранит в себе указатель на первый элмент этого массива.
Затем мы вызываем функцию modifySlice и передаем туда наш слайс s, а это значит, что мы передаем не копию значений, а копию указателя на базовый массив.
Из-за того, что мы напрямую меняем элемент слайса - i[0] = "3", то изменения уходят в этот базовый массив 
и элемент с индексом 0 меняется на "3". С этого момента все становится интереснее. Когда мы делаем append в слайс, происходит следующее:
так как массив не может изменить свой размер, ведь размер массива это вообще тип, то тут в дело вступает механика функции append - growSlice и создает новый массив
[6]string{"1", "2", "3", "", "", ""}, копируя значения старого массива, и добавление нового значения "4" уходит уже туда,
а в слайсе меняется указатель на этот новый массив.
Так, как слайс теперь указывает на массив [6]string, то следующее изменение по индексу - i[1] = "5", так же уходит в новый массив, как и последующий append.
После чего, функция modifySlice заканчивает работу, не возвращая этот новый слайс в main, поэтому мы получаем лишь 1 изменение в слайсе "s" - изменение элемента с индексом 0.

```
